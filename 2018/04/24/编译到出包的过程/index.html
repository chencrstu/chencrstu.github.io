编译到出包的过程

最新在折腾项目打包相关工作，包括framework，ipa的打包等，特记录如下。

从源码到ipa包：

  1. xcodebuild archive
    负责将源代码编译成xxx.app，根据实际情况加上-workspace, -project, -configuration, -archivePath等参数
  2. xcodebuild -exportArchive
    负责将xxx.app签名并打包成xxx.ipa

细化一下步骤：

  1. CompileC使用clang编译每个源代码文件
  2. LD将编译后的源代码文件进行链接操作，针对不同的机器架构生成不同的机器码
  3. lipo创建通用库
  4. ibtool处理storyboard，actool处理Assets文件
  5. dsymutil生成DSYM文件，其中存储函数地址映射，通过DSYM文件就可以通过崩溃的调用栈得到具体的函数位置，编译排查crash问题。
  6. codesign进行签名验证

一些命令:

  1. xcodebuild clean build CODE_SIGN_IDENTITY=”” CODE_SIGNING_REQUIRED=NO -target “${FRAMEWORK_NAME}” -configuration Release -arch arm64 -arch armv7 -arch armv7s only_active_arch=no defines_module=yes -sdk “iphoneos” | xcpretty
    生成只支持arm架构，不带签名信息的framework

  2. lipo -create -output ${all} ${arm64} ${x86}
    生成支持arm、x86架构的通用库

  3. dsymutil ${xxx.app} -o ${xxx.dDSYM}
    根据.app生成.dsym

  4. codesign -v ${xxx.app | xxx.framework}
    查看签名信息
